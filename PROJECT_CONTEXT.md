## 🔐 МЕХАНИЗМ E2E ШИФРОВАНИЯ (v8.2)

**Библиотека:** `tweetnacl` (подключается глобально).
**Алгоритм:** `nacl.secretbox` (Symmetric-key encryption).

### Генерация общего ключа (`getSharedKey`):
1.  Адреса собеседников сортируются лексикографически.
2.  Формируется строка: `chat-key-v1:${addr1}:${addr2}`.
3.  Пользователь подписывает эту строку своим MetaMask (`signer.signMessage`).
4.  Хеш подписи (`keccak256`) используется как симметричный ключ (32 байта).
5.  Ключ кэшируется в `Map sharedKeyCache` для текущей сессии.

### Шифрование (`encrypt`):
*   Генерируется случайный `nonce` (24 байта).
*   Текст шифруется ключом.
*   Результат: `Base64(nonce + ciphertext)`.

### Расшифровка (`decrypt`):
*   Извлекается `nonce` и `ciphertext` из Base64.
*   Используется тот же механизм генерации ключа (подпись той же строки).
*   Если ключ совпадает, сообщение расшифровывается.

> **Важно:** Безопасность зависит от того, что только владелец приватного ключа может сгенерировать правильный shared key через подпись.

## 🏗️ ПЛАН АРХИТЕКТУРНОГО РЕФАКТОРИНГА (BACKLOG)

> **Статус:** ⏸️ Отложено. Текущая монолитная архитектура `app.js` стабильна. Рефакторинг будет проведен итеративно при добавлении крупных фич (Звонки, Каналы).

### 🧐 Аудит текущего состояния `app.js`
Текущий файл (~1000 строк) функционален, но имеет следующие архитектурные ограничения:
1.  **Глобальное пространство имён:** Переменные `provider`, `signer`, `store` объявлены глобально, что рискует конфликтами при росте проекта.
2.  **Нарушение SRP (Single Responsibility Principle):** В одном файле смешаны Web3-логика, UI-рендеринг и криптография.
3.  **Сложность тестирования:** Жесткая привязка к DOM и глобальным переменным делает невозможным Unit-тестирование отдельных модулей.

### 🚀 Стратегия модуляризации (Future Steps)
При переходе к следующей мажорной версии (v9.0) планируется разбиение на 3 слоя:

#### 1. Папка `core/` (Фундамент)
*   `core/store.js` — Управление состоянием (контакты, настройки) вместо глобальных объектов.
*   `core/utils.js` — Утилиты (`escapeHtml`, валидация адресов).
*   `core/crypto.js` — Изоляция логики E2E шифрования (`tweetnacl`).

#### 2. Папка `web3/` (Блокчейн)
*   `web3/wallet.js` — Подключение MetaMask, переключение сетей.
*   `web3/contracts.js` — Абстракция вызовов Identity и MessageStorage.
*   `web3/scanner.js` — Логика сканирования событий для обнаружения чатов.

#### 3. Папка `ui/` (Интерфейс)
*   `ui/chatList.js` — Рендер списка чатов.
*   `ui/messages.js` — Рендер сообщений, скроллинг.
*   `ui/modals.js` — Управление модальными окнами.

#### 4. Точка входа `app.js`
Станет «тонким» файлом инициализации:
```javascript
import { initStore } from './core/store.js';
import { initWallet } from './web3/wallet.js';
import { initUI } from './ui/ui.js';

document.addEventListener('DOMContentLoaded', async () => {
    await initStore();
    await initWallet();
    initUI();
});
```
