## ✍️ ПОДПИСЬ СООБЩЕНИЙ КОШЕЛЬКОМ

1. Пользователь вводит текст.
2. Вызывается `signer.signMessage(text)` → подпись (65 байт).
3. Подпись сохраняется вместе с сообщением в контракте `MessageStorage`.
4. В UI отображается значок 🔐 (с возможностью верификации).

**Ключевой фрагмент из `app.js`:**
```javascript
async function signMessage(text) {
  if (!signer) throw new Error('Кошелёк не подключён');
  return await signer.signMessage(text);
}
```

## 🆕 ОТПРАВКА И ПОЛУЧЕНИЕ СООБЩЕНИЙ (ТЕКУЩАЯ РЕАЛИЗАЦИЯ)

### 📜 Контракт `MessageStorage.sol`

**Адрес:** `0x906DCA5190841d5F0acF8244bd8c176ecb24139D`  
**Функции:**
- `sendMessage(address recipient, string text, bytes signature)`
- `getConversation(address userA, address userB, uint256 startIndex, uint256 count)`

### 🔄 Процесс (на основе кода `sendMessage` и `loadMessagesForChat`)

1. Отправитель вызывает `sendMessage` с подписанным **открытым текстом**.
2. Транзакция майнится, сообщение сохраняется в блокчейне.
3. Получатель (или сам отправитель) вызывает `getConversation` с параметрами `startIndex=0, count=50`.
4. Сообщения объединяются, сортируются по времени и отображаются.
5. Каждые **10 секунд** (`startAutoRefresh`) происходит автообновление текущего чата.
6. Фоновое обнаружение новых чатов (`discoverNewChats`) выполняется каждые **30 секунд** путём перебора `messageCount` для всех известных адресов.
